피셔(성장주) 프레임으로 **한 기업(단건) 또는 3~5개 배치**에 대해 Deep Dive 진입 여부를 1~2시간 내 판정한다. 철학은 `PHILOSOPHY.md`, 파이프라인 위치는 `PIPELINE.md` 2단계.

**Shallow인 이유**: 훌륭함을 확신하려는 게 아니라 **Deep Dive에 1~2주를 투자할 가치가 있는가**만 판정. 확신(형용사 누적)은 Deep의 몫.

## 실행 모드

| 모드 | 용도 | 특징 |
| ------ | ------ | ------ |
| **단건** (기본) | 한 기업만 판단할 때 / 첫 실행 경험 쌓을 때 | Phase 0~10 중 Phase 9(배치 비교) 제외. Phase 10은 절대 기준 판정 |
| **배치** (3~5개 병렬) | Sensing 풀이 쌓여 월 1~2회 세션 | Phase 9 비교 테이블 생성. 상대 비교로 Keep 상태가 유의미해짐 |

입력 티커 수로 자연 분기. 호출 예시:

```text
단건:  "PHILOSOPHY.md, SHALLOW_DIVE.md를 읽고 {TICKER}에 대해 Shallow Dive를 실행해줘."
배치:  "PHILOSOPHY.md, SHALLOW_DIVE.md를 읽고 {T1}, {T2}, {T3}에 대해 Shallow Dive 배치를 실행해줘."
```

## Phase 1: 유니버스 하한선 체크 (게이트)

**입력**: 티커
**산출**: PASS/FAIL 판정 — **FAIL이면 즉시 탈락, Phase 2 이하 스킵**

피셔 프레임은 경영진 판단이 본체이므로 판단 가능한 규모·공시 수준이 전제 (`PHILOSOPHY.md` 섹션 6).

| 항목 | 조건 | 판정 | 근거 |
| ------ | ------ | ------ | ------ |
| 상장 기간 | ≥ 5년 | | |
| 컨퍼런스콜 | 정기 운영 + Q&A 전사본 공개 | | |
| 현 경영진 재직 | ≥ 3년 | | |
| 임원 이직률 파악 | DART/EDGAR/링크드인으로 추적 가능 | | |
| 업계 평판 접근 | 최소 한 단계 거쳐 닿음 | | |

하나라도 FAIL → "피셔 프레임 판단 불가, 능력 범위 밖" → **탈락** 처리.

## Phase 3: 정량 기초 3년

**입력**: Gold 패널
**산출**: 3년 추이 표 + 한 줄 코멘트

피셔식 정밀 계산은 Deep Stream C. 여기는 일관성 vs 변동성만 본다.

| 지표 | 계산 | 해석 포인트 |
| ------ | ------ | ------------ |
| 매출 | revenue_ttm | 추이·성장률 |
| 영업이익률 | EBIT_ttm / Revenue_ttm | 마진 유지/개선 |
| FCF | CFO_ttm − CAPEX_ttm | 일관성 |
| ROIC | EBIT × (1−0.22) / (총자산 − 유동부채 − 현금) | 자본효율성 |
| R&D / 매출 | R&D_ttm / Revenue_ttm | 재투자 강도 |

**3년 추세 한 줄 코멘트** 필수: "일관되게 성장" / "변동성 큼" / "최근 둔화" 등.

## Phase 6: Fisher 15 정량 5개

**입력**: Phase 3 정량 기초 + 경영진 가이던스
**산출**: 5개 조건 판정 표

Fisher 15 중 **공개 숫자로 답할 수 있는 5개**. 정성 10개는 Deep Stream B. `knowledge/성장주_투자.md` 원전 참조.

| # | 기준 | 지표 | 판정 | 근거 |
| --- | ------ | ------ | ------ | ------ |
| 1 | 매출 성장 잠재력 | 3년 매출 CAGR + 경영진 가이던스 | PASS/MIXED/FAIL | |
| 3 | R&D 투입 대비 성과 | R&D/매출 추이 + 매출 성장 연동 | | |
| 5 | 충분한 이익률 | 영업이익률 vs 업계 평균 | | |

## Phase 10: 결과 판정

**입력**: Phase 1~9 전체
**산출**: 4갈래 중 하나 + 확신도

| 결과 | 조건 | 다음 행동 |
| ------ | ------ | ---------- |
| **Deep Dive 진입** | 유니버스 PASS + 사업모델 명료 + 정량 5개 중 PASS ≥ 3 + (배치면 상대 우위 명확) | 동시 Deep 2~4개 한도 내 착수 |
| **Keep (다음 배치 비교용)** | 매력적이나 Deep 우선순위는 아직 — 다음 배치에서 신규 후보와 **상대 비교**하면 판단이 날카로워질 상태 | 다음 Shallow Batch에 자동 포함 |
| **Parking Lot** | 분기 재방문이 맞는 상태 — 외부 이벤트 대기(신제품 출시, 규제 결론, 사이클 저점 등) | 분기 리뷰 세션 |
| **탈락** | 유니버스 FAIL / 2분 테스트 실패 / 명확한 결격 | Sensing 로그에서도 제거 고려 |

### Keep vs Parking 구분

> **"다음 배치 비교가 지금 유의미한가?"** 가 유일한 기준.
>
> - **Keep** — 현재 알고 있는 것만으로 상대 비교에 정보가치 있음
> - **Parking** — 대기해야 할 외부 이벤트 있거나 상대 비교의 문제가 아닌 내부 전환 대기

Keep이 **2~3 배치 연속 이월되면 Parking 강등** 또는 **Deep 진입 재판단**.

### 단건 모드의 Keep 주의

단건은 비교 대상이 없으므로 Keep의 원래 효용(상대 비교)이 없음. 단건에서 Keep 결정은 "다음 배치에 올려 비교하겠다"는 **예약** 개념으로만 사용 — 이는 실질적으로 **배치를 기다리는 Parking-like 상태**와 구분하기 어려움.

**권고**: 단건 모드에서는 3갈래(Deep 진입 / Parking / 탈락)로 단순화. Keep은 배치 모드에서만.

### 확신도 명시

| 확신도 | 의미 |
| -------- | ------ |
| 높음 | 판정 근거를 한 문단으로 설명 가능 |
| 중간 | 방향은 있으나 근거 부족 — Keep 기본값(배치 모드) |
| 낮음 | 이 실행에서는 판단 보류 — Parking |

**낮음에서 Deep 진입은 금지.**

## 원칙

1. **Shallow에서 확신 만들려 하지 말 것** — Deep 진입 여부만 판정
2. **유니버스 하한선은 게이트** — 미통과 = 피셔 프레임 판단 불가 = 탈락. "어떻게든 해보자" 금지
3. **병렬의 이점은 비교** — 배치가 단건보다 Deep 진입 판단에 더 날카로움. 단건은 빠르지만 Keep의 의미가 약해짐
4. **Sensing 오염 금지** — Shallow 결과가 Sensing 로그의 기록 방식을 바꾸지 않도록 (Sensing은 평가 없이 기록만)
5. **프레임 미정 허용** — shallow에서 버핏형/피셔형 확정 못 해도 됨. Deep 진입 전까지만 확정
6. **정량 5개는 축소판이지 결론 아님** — FAIL 많아도 Deep에서 재평가 가능. 단 우선순위 하향
7. **Keep과 Parking은 다른 상태** — Keep은 배치 비교용, Parking은 분기 재방문
8. **확신도 낮음 = Deep 진입 금지** — Parking
9. **AI 협업의 1~2시간 compression을 확신 형성으로 착각하지 말 것** — 본격 확신은 Deep + 숙성기
